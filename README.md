## Цикломатска комплексност

Цикломатската комплексност покажува колку независни патеки има низ една функција. Ја пресметав со формулата:

`V(G) = P + 1`

каде што `P` е бројот на предикатни јазли во функцијата. Како предикатни јазли ги бројам `if` условите и `for` циклусите, бидејќи тие прават разгранување во текот на извршување.

Може да се користи и формулата:

`V(G) = E - N + 2`

каде што `E` е бројот на ребра, а `N` е бројот на јазли во Control Flow Graph-от. Во оваа документација ја користам формулата `P + 1`, бидејќи е поедноставна за овие две функции.

Важно: сложените услови со `&&` и `||` тука ги бројам како еден предикатен јазол, затоа што се наоѓаат во еден `if` услов. Комбинациите на подусловите се обработени подолу во делот за Multiple condition критериумот.

### searchBookByTitle

За функцијата `searchBookByTitle` цикломатската комплексност е 5.

Предикатни јазли се:

1. `if (title.isEmpty())`
2. `for (Book book : books)`
3. `if (book.getTitle().equalsIgnoreCase(title) && !book.isBorrowed())`
4. `if (results.isEmpty())`

Затоа:

`P = 4`

`V(G) = P + 1 = 4 + 1 = 5`

Ова значи дека функцијата има 5 независни патеки. Тие произлегуваат од проверката за празен наслов, циклусот низ книгите, условот за додавање книга во резултатите и проверката дали резултатите се празни.

### borrowBook

За функцијата `borrowBook` цикломатската комплексност е 5.

Предикатни јазли се:

1. `if (title.isEmpty() || author.isEmpty())`
2. `for (Book book : books)`
3. `if (book.getTitle().equalsIgnoreCase(title) && book.getAuthor().equalsIgnoreCase(author))`
4. `if (!book.isBorrowed())`

Затоа:

`P = 4`

`V(G) = P + 1 = 4 + 1 = 5`

Ова значи дека функцијата има 5 независни патеки. Тие произлегуваат од проверката за невалиден влез, циклусот низ книгите, проверката дали книгата се совпаѓа со бараниот наслов и автор, и проверката дали книгата е веќе изнајмена.
